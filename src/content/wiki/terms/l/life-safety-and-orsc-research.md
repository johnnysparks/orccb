## Definition

Life safety and ORSC research is the practice of finding, interpreting, and documenting Oregon Residential Specialty Code requirements that protect occupants and responders: egress, smoke/CO alarms, guards, stairs, structural safety, fire separation, and related systems.

## Why it matters

Life-safety errors can stop permits, fail inspections, and create serious legal and ethical risk. Competent GCs do not rely on memory alone; they research code sections, record decisions, and coordinate details before work is covered.

## Where it appears in a real project

- Permit drawings and code notes.
- Framing, stair, and guard layout verification.
- Alarm device placement and electrical coordination.
- Final life-safety inspections and certificate of occupancy milestones.

## Inputs needed

- Current adopted ORSC edition and local guidance.
- Project scope (new, addition, alteration, repair).
- Occupancy/use assumptions and bedroom counts.
- Plan set with dimensions and sections.
- Jurisdiction correction comments.

## Outputs produced

- Code research log with section citations.
- Marked-up plans showing life-safety compliance points.
- RFI/questions list for unclear interpretation.
- Field verification checklist tied to inspections.

## Key decisions

1. Which ORSC provisions govern this scope type.
2. Where alternate details may be needed due to existing constraints.
3. Which life-safety checks become mandatory hold points.
4. When to seek clarification from building officials.

## Basic workflow

1. Define project scope and code path.
2. Build a life-safety topic list (egress, stairs/guards, alarms, fire separation, etc.).
3. Research each topic in ORSC and document section references.
4. Translate requirements into drawing notes and field tolerances.
5. Review with trades before rough-in/cover.
6. Verify in field and capture photo/document evidence for inspection readiness.

## Risks and edge cases

- Using outdated code-cycle references.
- Conflicting internet summaries instead of adopted text.
- Existing building conditions that complicate strict prescriptive paths.
- Late plan changes that unintentionally break egress or alarm layouts.

## Oregon/Ashland relevance

Oregon projects must follow the currently adopted residential code framework and jurisdiction procedures. Ashland teams should verify local process expectations early, then maintain a clean correction-response package so life-safety issues are resolved quickly.

## Example scenario

During permit corrections, a learner-GC finds stair geometry and smoke alarm notes under-documented. They create an ORSC research log, revise sheet notes, and align electrical scope before rough-in. Inspection passes without re-opened walls.

## Checklist

- [ ] Current ORSC adoption confirmed.
- [ ] Life-safety code topics listed for project scope.
- [ ] Section-by-section research log completed.
- [ ] Plan notes and details updated from research.
- [ ] Trade coordination completed before rough-in.
- [ ] Inspection hold-point checklist used in field.

## Quiz questions

1. What is the best source for life-safety requirements on an Oregon residential project?  
   A) Social media summaries.  
   B) Current adopted ORSC text and jurisdiction guidance.  
   C) Old project notes only.  
   D) Manufacturer brochures alone.

2. Why keep a code research log?  
   A) It slows production intentionally.  
   B) It documents decisions and supports permit/inspection responses.  
   C) It replaces drawings.  
   D) It is only needed on commercial work.

3. What is a common failure mode?  
   A) Verifying adoption dates.  
   B) Coordinating alarms with electrical scope.  
   C) Using outdated code references.  
   D) Reviewing corrections quickly.

**Answer key:** 1-B, 2-B, 3-C.
