## Definition

**Exterior change review triggers** are the specific project conditions that cause additional planning/design review or expanded permit scrutiny when exterior residential work is proposed.

In training terms, this means identifying which planned exterior actions (height, footprint, facade changes, tree impacts, driveway/streetscape shifts, visible massing changes) move a project from simple permit flow into broader review.

## Why it matters

- Missing a trigger can invalidate schedule commitments and contract assumptions.
- Trigger awareness helps prevent "permit surprise" change orders.
- It improves permit package completeness and revision readiness.
- It protects owner confidence through clear early risk communication.

## Where it appears in a real project

- Scope definition during estimating.
- Pre-application planning calls.
- Exterior design development and value engineering.
- Permit package quality-control review.
- Midstream changes requested by owner after drawings are near final.

## Inputs needed

- Existing and proposed exterior conditions (site plan, elevations, sections).
- District applicability and compatibility requirements.
- Jurisdiction-specific review threshold information and intake guidance. Source: ors-chapter-455, oregon-bcd-jurisdiction-renewals-oar-918-020.
- Oregon code administration context references. Source: oregon-bcd-adopted-codes-online.
- Owner priorities for cost, timeline, and neighborhood fit.

## Outputs produced

- Trigger log: what is triggered, why, and what review path is required.
- Updated permit sequence map and expected review durations.
- Scope options showing lower-trigger and higher-trigger alternatives.
- Owner decision record with timeline and risk implications.

## Key decisions

1. Which exterior changes are essential versus optional for owner goals?
2. Is a lower-trigger design variant acceptable to preserve timeline?
3. Should the project pause for pre-application input before full permit package assembly?
4. How much contingency is needed for comments, hearings, or revisions?

## Basic workflow

1. Inventory all proposed exterior changes from concept drawings.
2. Crosswalk each change against known local trigger thresholds.
3. Classify each item: no added review, potential trigger, confirmed trigger.
4. Validate ambiguous items with planning/building staff.
5. Revise design/sequence strategy and owner expectations.
6. Submit with supporting narrative that addresses triggered concerns.

## Risks and edge cases

- Minor owner-requested facade edits unintentionally trigger higher review.
- Combined small changes create a trigger even when each appears minor alone.
- Conflicting advice from informal conversations vs written agency guidance.
- Trade sequencing assumes permit timing that no longer applies after trigger confirmation.

## Oregon/Ashland relevance

Because Oregon projects operate in a statewide code framework with local implementation, trigger management depends on confirming current local process details for the property rather than relying on assumptions from prior jobs. Source: ors-chapter-455, oregon-bcd-adopted-codes-online, oregon-bcd-jurisdiction-renewals-oar-918-020.

For Ashland-area work, visible exterior modifications and neighborhood-context sensitivity can elevate review complexity, so trigger checks should happen before final pricing.

## Example scenario

A homeowner asks for a "small" second-story bump-out during design development. The GC's trigger check shows the added exterior massing likely changes review complexity in that district. The team prepares two paths: (A) proceed with bump-out and longer review window, or (B) rework interior layout to avoid the trigger. The owner chooses Path B to protect move-in date.
