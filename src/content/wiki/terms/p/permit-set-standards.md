## Definition

Permit set standards are the minimum completeness, clarity, and coordination benchmarks a drawing/spec package should meet before jurisdiction submittal.

## Why it matters

Poor permit packages create avoidable correction cycles, schedule slip, and team confusion. Clear standards improve first-pass acceptance and create a better construction baseline.

## Where it appears in a real project

- Internal pre-submittal QC.
- Permit intake and correction responses.
- IFC (issued-for-construction) alignment after permit approval.

## Inputs needed

- Current jurisdiction intake checklist.
- Coordinated plan, section, detail, and schedule sheets.
- Code summary assumptions.
- Structural and MEP supporting documents as required.

## Outputs produced

- Permit-ready sheet set and supporting documents.
- Submittal transmittal and revision log.
- Correction-response matrix with responsibility and due dates.

## Key decisions

1. What level of detail is mandatory at first submittal.
2. Which coordination checks are “hard gates” before upload.
3. Who owns correction responses and deadline tracking.
4. How to align permit-approved set with field issue set.

## Basic workflow

1. Build a jurisdiction-specific permit checklist.
2. Run interdisciplinary sheet QA (dimensions, references, notes, symbols).
3. Confirm code summary consistency.
4. Submit with clean naming/version control.
5. Triage corrections and issue targeted revisions.
6. Publish permit-approved baseline to production teams.

## Risks and edge cases

- Sheet reference loops and missing detail callouts.
- Inconsistent dimensions across plan/elevation/section.
- Untracked revision uploads causing field confusion.
- Intake rejection due to formatting/admin omissions.

## Oregon/Ashland relevance

Oregon jurisdictions can vary in intake mechanics and correction workflows. For Ashland work, a disciplined permit-set QA process helps reduce turnaround surprises and keeps project start dates more predictable.

## Example scenario

A trainee GC introduces a 30-point pre-submittal checklist. On the next project, they catch missing structural detail references and inconsistent window notes before upload, cutting one full correction cycle.

   D) Avoiding code references.

2. Which practice best reduces correction rounds?  
   A) Submit early without QA.  
   B) Use a structured pre-submittal checklist.  
   C) Ignore reference consistency.  
   D) Upload unversioned files.

3. Why is version control critical after permit approval?  
   A) It is optional if teams are small.  
   B) It prevents field teams from building from outdated sheets.  
   C) It only matters for architects.  
   D) It replaces transmittals.
